# ScreenKite 功能增强需求规格说明书（SPEC）

> **版本**: v1.0  
> **日期**: 2026-03-24  
> **作者**: ScreenKite Core Team  
> **状态**: 已批准  

---

## 1. 概述

### 1.1 背景

ScreenKite 是一款 macOS 原生屏幕录制工具，基于 Swift + ScreenCaptureKit + Metal 构建，主打自动运镜和高速导出。目前处于 Beta 阶段，完全免费、无水印、无订阅。

用户调研显示，以下三个功能缺失严重影响了使用体验，是 Beta → v1.0 过渡期间必须优先实现的功能：

1. **定时录制（Scheduled Recording）**
2. **全局快捷键（Global Hotkeys）**
3. **静默录制/DND 模式（Do Not Disturb Recording）**

### 1.2 目标

在不破坏现有核心功能的前提下，以**低侵入性**方式为 ScreenKite 添加三个高价值功能，使产品达到 v1.0 候选版本标准。

### 1.3 范围

| 功能 | 优先级 | 目标用户增量 |
|------|--------|------------|
| 定时录制 | P1 | +30% 留存率 |
| 全局快捷键 | P1 | +15% 操作效率 |
| 静默录制模式 | P1 | +20% 专业体验 |

## 2. 功能需求

### 2.1 功能一：定时录制（Scheduled Recording）

#### 2.1.1 功能描述

用户可以预先设定录制开始时间，支持一次性录制和周期性录制（按周/按天）。系统到达预设时间后自动开始录制，无需人工干预。

#### 2.1.2 用户故事

| ID | 角色 | 故事 |
|----|------|------|
| US-001 | 在线教师 | 作为教师，我希望每周一 9:00 自动录制课程，这样我不用每次手动开始 |
| US-002 | 企业培训师 | 作为培训师，我希望设定今天下午 3:00 开始录制例会，这样我在会议开始时就已经在录制了 |
| US-003 | 内容创作者 | 作为博主，我希望每天固定时间录制我的工作过程，无需分心去按录制键 |

#### 2.1.3 使用场景

**场景 1：周期性录制**
```
触发条件：用户设置了"每周一 9:00"录制任务
前置条件：ScreenKite 在后台运行，已选择录制范围和音频来源
执行流程：
  1. 每周一 9:00 前 1 分钟，系统通知提醒"即将开始录制"
  2. 9:00 整，ScreenKite 自动启动录制
  3. 录制过程中，菜单栏图标显示录制状态（红点）
  4. 用户按 F9 或点击菜单栏图标停止录制
  5. 录制完成，自动保存到用户指定的输出目录
```

**场景 2：一次性录制**
```
触发条件：用户设置了"今天 15:00"一次性录制任务
前置条件：ScreenKite 在后台运行
执行流程：
  1. 14:59 系统通知提醒"录制将在 1 分钟后开始"
  2. 15:00 自动启动录制
  3. 用户主动停止
  4. 任务自动删除（一次性任务）
```

#### 2.1.4 详细需求

| ID | 需求描述 | 类型 | 验收标准 |
|----|---------|------|---------|
| F1-01 | 支持设置具体日期和时间的定时录制 | 功能 | 在指定时间 ±5 秒内启动录制 |
| F1-02 | 支持按周重复（选择具体星期几） | 功能 | 每周指定日期自动触发 |
| F1-03 | 支持按天重复（每日固定时间） | 功能 | 每个自然日自动触发 |
| F1-04 | 录制开始前 1 分钟推送系统通知 | 功能 | 通知在录制开始前 60 秒 ±5 秒发送 |
| F1-05 | 支持查看/编辑/删除已创建的定时任务 | 功能 | 所有操作在 UI 上可见且可执行 |
| F1-06 | 定时录制期间 ScreenKite 崩溃后自动恢复 | 容错 | 崩溃重启后检测到定时录制过期，自动跳过 |
| F1-07 | 定时录制任务持久化存储（重启后保留） | 功能 | 应用重启后所有定时任务仍然存在 |
| F1-08 | 支持设置录制持续时间上限（超时自动停止） | 功能 | 达到上限后自动停止录制 |

#### 2.1.5 界面需求

- **设置面板**：菜单栏图标 → 下拉菜单 → "定时录制..." 选项
- **任务列表**：显示所有定时任务（名称、时间、状态）
- **新建任务表单**：日期选择器 + 时间选择器 + 重复选项 + 录制参数（范围、音频）
- **通知界面**：系统原生通知，支持点击打开 ScreenKite 主窗口

#### 2.1.6 数据模型

```
ScheduledTask:
  - id: UUID
  - name: String                    // 任务名称（如"周一课程录制"）
  - triggerType: TriggerType        // .once / .daily / .weekly
  - scheduledDate: Date             // 下次触发时间
  - repeatDays: [Int]?              // 星期几重复（1=周一, 7=周日），仅 weekly
  - recordingSettings: RecordingConfig  // 录制参数
  - durationLimit: TimeInterval?    // 最大录制时长（秒），nil 表示无限制
  - isEnabled: Bool                 // 是否启用
  - createdAt: Date
  - lastTriggeredAt: Date?
  - nextTriggerDate: Date?          // 计算后的下次触发时间

RecordingConfig:
  - captureType: CaptureType       // .fullScreen / .window / .region
  - targetWindow: NSWindow?         // 指定窗口（仅 window 类型）
  - captureRegion: CGRect?          // 指定区域（仅 region 类型）
  - includeAudio: Bool
  - includeMicrophone: Bool
  - frameRate: Int                  // 帧率
```

#### 2.1.7 技术约束

- **最小支持版本**：macOS 12.0（Monterey）
- **定时精度**：±5 秒
- **后台运行**：需要勾选"登录项"并获得屏幕录制权限
- **定时机制**：使用 macOS `DispatchSourceTimer`，结合 `launchd` 复活机制
- **通知**：使用 `UserNotifications` 框架

### 2.2 功能二：全局快捷键（Global Hotkeys）

#### 2.2.1 功能描述

在 ScreenKite 录制过程中，用户无需切换到 ScreenKite 窗口，直接通过全局键盘快捷键控制录制操作。

#### 2.2.2 用户故事

| ID | 角色 | 故事 |
|----|------|------|
| US-004 | 技术博主 | 作为博主，我希望用 F9 开始录制、F10 停止，这样我可以在任何应用窗口下操作 |
| US-005 | 在线教师 | 作为教师，我希望用 F11 快速添加时间戳标记，这样我可以在讲课过程中快速标记重要时刻 |
| US-006 | 会议记录者 | 作为行政，我希望用空格键暂停/继续录制，这样我在接到电话时可以快速暂停 |

#### 2.2.3 详细需求

| ID | 需求描述 | 类型 | 验收标准 |
|----|---------|------|---------|
| F2-01 | F9 全局快捷键：开始/恢复录制 | 功能 | 在任意应用前台均可触发 |
| F2-02 | F10 全局快捷键：暂停/恢复录制 | 功能 | 暂停状态有明确视觉反馈 |
| F2-03 | F11 全局快捷键：停止录制 | 功能 | 停止后弹出导出面板 |
| F2-04 | F12 全局快捷键：添加时间戳标记 | 功能 | 视频时间轴上出现标记点 |
| F2-05 | 支持自定义快捷键绑定 | 功能 | 用户可在设置中修改绑定 |
| F2-06 | 快捷键冲突检测与提示 | 功能 | 与系统快捷键冲突时报错 |
| F2-07 | 录制状态下菜单栏图标变化 | 功能 | 红点 + 录制时长显示 |
| F2-08 | 快捷键在非录制状态下不响应 | 安全 | 避免误触发 |

#### 2.2.4 默认快捷键映射

| 快捷键 | 操作 | 条件 |
|--------|------|------|
| F9 | 开始/恢复录制 | 非录制状态 |
| F10 | 暂停录制 | 录制中 |
| F10 | 恢复录制 | 暂停状态 |
| F11 | 停止录制 | 录制中或暂停中 |
| F12 | 添加时间戳标记 | 录制中 |

#### 2.2.5 技术实现

- **框架**：`Carbon.HotKey`（基于 Carbon Events API）或 `MASShortcut`
- **冲突检测**：使用 `CGEvent.tapCreate` 拦截并检测快捷键
- **实现方式**：注册全局 `EventHotKey`，在后台运行
- **权限**：需要"辅助功能"权限（Accessibility）

### 2.3 功能三：静默录制/DND 模式（Do Not Disturb Recording）

#### 2.3.1 功能描述

一键开启"勿扰录制模式"，自动开启 macOS Focus（DND），录制结束后自动恢复原有设置。防止录制过程中弹出通知打扰。

#### 2.3.2 用户故事

| ID | 角色 | 故事 |
|----|------|------|
| US-007 | 远程面试官 | 作为 HR，我希望录制面试过程时不被通知打扰，这样面试视频是干净专业的 |
| US-008 | 游戏主播 | 作为主播，我希望录制游戏实况时通知不弹出来，这样游戏过程不会被遮挡 |
| US-009 | 知识博主 | 作为博主，我希望录制演示时我的隐私通知不会被录进去，保护我的隐私 |

#### 2.3.3 详细需求

| ID | 需求描述 | 类型 | 验收标准 |
|----|---------|------|---------|
| F3-01 | 一键开启勿扰录制模式 | 功能 | 点击后自动开启 DND，无需手动操作 |
| F3-02 | 录制结束后自动恢复 DND 状态 | 功能 | 若录制前 DND 为关闭状态，录制后也关闭 |
| F3-03 | 录制结束后保留原 Focus 状态 | 功能 | 录制前保存，录制后恢复 |
| F3-04 | 紧急取消：录制中随时可关闭 DND | 功能 | 不影响录制进程 |
| F3-05 | DND 图标状态指示 | UI | 菜单栏显示 DND 图标 |
| F3-06 | 录制时间过长时 DND 自动续期 | 容错 | 录制超过 2 小时，续期 DND |

#### 2.3.4 状态机

```
IDLE:
  用户点击"开始勿扰录制"
  → 保存当前 Focus 状态 → 开启 DND → 开始录制 → RECORDING

RECORDING:
  - 用户停止录制 → 停止录制 → 恢复 Focus → IDLE
  - 录制超时 → 停止录制 → 恢复 Focus → IDLE
  - 应用崩溃 → 系统自动关闭 DND（macOS 行为）

用户主动取消 DND：
  → 保持录制 → 显示警告"通知可能会被录入" → RECORDING
```

#### 2.3.5 技术实现

- **Focus API**：`IFFocusStatus`（macOS 12+，通过 Private Framework）或 `NSUserNotificationCenter`
- **兼容方案**：若 Focus API 不可用，使用 `defaults write` 操作 macOS DND 配置
- **隐私保护**：录制前检测并模糊处理通知区域

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 定时录制启动延迟 | ≤ 5 秒 |
| 快捷键响应时间 | ≤ 100ms |
| DND 模式切换时间 | ≤ 2 秒 |
| 应用冷启动时间 | ≤ 3 秒 |
| 内存占用（空闲状态） | ≤ 80MB |
| 内存占用（录制中） | ≤ 300MB |

### 3.2 兼容性

| 项目 | 要求 |
|------|------|
| macOS 最低版本 | macOS 12.0 Monterey |
| macOS 测试版本 | 12.x, 13.x, 14.x, 15.x |
| Apple Silicon | 原生支持（arm64） |
| Intel Mac | 原生支持（x86_64） |

### 3.3 安全与隐私

- 不收集任何用户录制内容
- 定时任务数据仅存储在本地
- 快捷键注册不涉及网络通信
- 不请求网络权限（本次三个功能均无需网络）

### 3.4 可维护性

- 使用 XcodeGen 管理项目配置
- 全部采用 Swift 5.9+
- 遵循 Swift API Design Guidelines
- 所有新功能有独立测试覆盖

## 4. 验收标准

### 4.1 定时录制

- [ ] 设置 1 分钟后的定时录制，精确在指定时间启动
- [ ] 设置每周一 9:00 的周期性任务，连续 4 周正确触发
- [ ] 应用重启后定时任务仍然存在
- [ ] 录制前 1 分钟收到系统通知
- [ ] 删除定时任务后不再触发

### 4.2 全局快捷键

- [ ] 在 Safari 窗口前台按下 F9，成功开始录制
- [ ] 录制状态下按下 F10，录制暂停
- [ ] 暂停状态下按下 F10，录制恢复
- [ ] 录制中按下 F12，时间轴出现标记
- [ ] 自定义快捷键成功生效
- [ ] 与系统快捷键冲突时报错

### 4.3 静默录制

- [ ] 点击"勿扰录制"，DND 自动开启（可从控制中心确认）
- [ ] 停止录制后，DND 恢复到录制前状态
- [ ] 录制中手动关闭 DND，录制继续但给出警告
- [ ] 录制 2 小时后 DND 自动续期
- [ ] 应用崩溃后 DND 由 macOS 系统自动关闭

## 6. 风险登记

| 风险 | 影响 | 可能性 | 缓解策略 |
|------|------|--------|---------|
| Focus API 在 macOS 13+ 有行为变更 | 高 | 中 | 提供手动 DND 开关作为降级方案 |
| 全局快捷键与第三方应用冲突 | 中 | 低 | 提供自定义绑定，添加冲突检测 |
| 定时录制在系统休眠后失效 | 中 | 中 | 结合 launchd 复活机制 |
| ScreenCaptureKit 在 macOS 12 系统不稳定 | 中 | 低 | 最低版本提升至 macOS 13 |
